# URLスライドショーアプリケーション

URLのリストを順番に自動表示するWebアプリケーションです。

## デモ

**GitHub Pages**: [https://kwu-cms.github.io/url-slideshow/](https://kwu-cms.github.io/url-slideshow/)

## 機能

- **URL管理**: URLの追加（複数行対応）、削除、順序変更
- **自動表示**: 設定した時間ごとにURLを順番に表示
- **ループ再生**: 最後のURLまで表示したら最初に戻る機能
- **フルスクリーン表示**: ウィンドウ内で全面表示モードに切り替え
- **データ管理**: URLリストのエクスポート/インポート（JSON形式）
- **自動保存**: ブラウザのローカルストレージに自動保存
- **URLパラメータ対応**: URLパラメータから設定を読み込んで自動再生可能
- **再生用URLコピー**: 現在の設定をURLパラメータ形式でコピー

## 使用方法

### 1. アプリケーションの起動

`index.html`をブラウザで開いてください。

### 2. URLの追加

1. 「URLを追加（複数行対応）」欄にURLを入力
   - 複数のURLを改行区切りで一度に追加可能
   - `http://`または`https://`がない場合は自動的に`https://`が追加されます
2. 「追加」ボタンをクリック、または`Ctrl+Enter`（Macでは`Cmd+Enter`）を押す
3. URLがプレイリストに追加されます

**初期状態**: URLが追加されていない場合は「URLがありません」と表示されます。

### 3. 表示設定

- **表示時間**: 各URLを表示する時間（秒単位）を設定（デフォルト: 5秒）
- **ループ再生**: チェックを入れると、最後のURLまで表示したら最初に戻ります（デフォルト: オン）

### 4. URLプレイリストの管理

URLプレイリストは、できる限り多くの項目を一度に表示できるように最適化されています。項目数が多い場合はスクロールバーで表示できます。

- **順序変更**: ↑/↓ボタンでURLの順序を変更
- **削除**: 「削除」ボタンでURLを削除（確認ダイアログが表示されます）
- **現在のURL**: 再生中は現在表示中のURLがハイライト表示されます

### 5. スライドショーの開始

1. 「開始」ボタンをクリック、またはスペースキーを押す
2. URLが順番に表示されます
3. 「停止」ボタンで停止できます

### 6. フルスクリーン表示

「フルスクリーン」ボタンをクリックすると、ウィンドウ内で全面表示モードに切り替わります。
フルスクリーン時は、画面右下にコントロールパネルが表示されます。

- フルスクリーン時に自動的に再生が開始されます（URLが設定されている場合）
- フルスクリーン解除時は自動的に停止します
- フルスクリーン解除は、コントロールパネルの「×」ボタンで行います

### 7. データのエクスポート/インポート

- **エクスポート**: 「エクスポート」ボタンをクリックすると、現在のURLリストと設定をJSONファイルとしてダウンロードします
- **インポート**: 「インポート」ボタンをクリックして、以前エクスポートしたJSONファイルを選択すると、URLリストと設定を復元できます

### 8. 再生用URLのコピー

「再生用のURLをコピー」ボタンをクリックすると、現在の設定をURLパラメータ形式でクリップボードにコピーします。
このURLを開くと、設定が自動的に読み込まれ、フルスクリーンモードで再生が開始されます。

### 9. 設定のクリア

「設定をクリア」ボタンをクリックすると、すべての設定を初期状態に戻します。
- URLリストがクリアされます
- 表示時間が5秒にリセットされます
- ループ再生がオンにリセットされます
- ローカルストレージのデータも削除されます

### 10. 操作方法の確認

「?」ボタンをクリックすると、操作方法のモーダルが表示されます。

## キーボードショートカット

- **スペースキー**: スライドショーの開始/停止（入力欄にフォーカスがある場合は無効）
- **Ctrl+Enter**（Macでは**Cmd+Enter**）: URL追加

## URLパラメータ機能

URLパラメータを使用して、設定を読み込んで自動再生することができます。

### パラメータ一覧

- `urls`: URLリスト（カンマ区切り、URLエンコードが必要）
- `displayTime`または`time`: 表示時間（秒）
- `loop`または`looping`: ループ再生（`true`/`false`または`1`/`0`）
- `fullscreen`: フルスクリーンモード（`true`/`false`または`1`/`0`）
- `playing`または`play`または`state`: 自動再生（`true`/`false`または`1`/`0`または`playing`）

### 使用例

```
index.html?urls=https://example.com,https://example2.com&displayTime=10&loop=true&fullscreen=true&playing=true
```

このURLを開くと、指定したURLが設定され、10秒間隔でループ再生され、フルスクリーンモードで自動的に再生が開始されます。

## ブラウザ対応

モダンブラウザ（Chrome、Firefox、Safari、Edge）で動作します。

## 注意事項

- 一部のWebサイトは、iframeでの表示を制限している場合があります（X-Frame-Optionsヘッダーなど）
- そのようなサイトは、新しいタブで開くか、直接アクセスしてください
- ローカルストレージのデータは、ブラウザの設定によってクリアされる場合があります
- URLパラメータで設定を読み込む場合、ローカルストレージの設定より優先されます

## GitHub Pagesでのデプロイ

このプロジェクトはGitHub Pagesでホスティングできます。

### 自動デプロイ（推奨）

GitHub Actionsを使用して自動デプロイが設定されています。`main`ブランチにpushするだけで、自動的にGitHub Pagesにデプロイされます。

初回設定：
1. リポジトリの設定（Settings）に移動
2. 「Pages」セクションを開く
3. 「Source」で「GitHub Actions」を選択
4. 「Save」をクリック

これで、`main`ブランチにpushするたびに自動的にデプロイされます。

### 手動デプロイ

GitHub Actionsを使わない場合：
1. リポジトリの設定（Settings）に移動
2. 「Pages」セクションを開く
3. 「Source」で「Deploy from a branch」を選択
4. 「Branch」で「main」を選択し、「/ (root)」を選択
5. 「Save」をクリック

数分後、`https://kwu-cms.github.io/url-slideshow/` でアクセスできるようになります。
